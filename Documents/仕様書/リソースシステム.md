# リソースシステム仕様書

本作のリソースは「固体」と「液体」の2つの物理形態に分類され、それぞれ異なる保管・輸送ルールを持ちます。

## 1. リソースの分類

### 1.1 固体 (Solid Resources)

- **特徴**: 個別の単位（アイテム）として存在し、地面に落ちたりチェストに格納されたりする。
- **輸送**: プレイヤーまたは「運搬」設定の牛が物理的に持ち運ぶ。
- **例**: 栄養ペレット（飼料）、チーズ、バター、木材、資材。
- **保管**:
  - **チェスト**: 1つのチェストには1種類のリソースのみ保管可能。容量制限あり。

### 1.2 液体 (Liquid Resources)

- **特徴**: 連続的な量として存在し、施設間をパイプで移動する。量と「品質」の概念を持つ。
- **輸送**: パイプネットワークを通じて自動的に移動する。
- **例**: 牛乳。
- **保管**:
  - **タンク**: 液体を溜めるための施設。容量制限あり。

## 2. 液体の物理挙動

### 2.1 混合ロジック（平均品質の更新）

タンクに新たに液体 $V_{new}$（品質 $Q_{new}$）が流入すると、タンク内の平均品質 $Q_{avg}$ は以下の式で即座に更新されます。

$$Q_{avg} = \frac{(V_{old} \times Q_{old}) + (V_{new} \times Q_{new})}{V_{old} + V_{new}}$$

- $V_{old}, Q_{old}$: 流入前のタンク内の量と品質。
- タンクが空の場合、最初に流入した液体の品質がそのまま平均品質となります。

### 2.2 流量制限

- パイプライン（施設A→施設B）を流れる液体には流量上限があります。
- **基本流量**: 1 tick あたり 1 単位（設定により調整可能）。
- 複数のパイプが接続されている場合でも、各接続（エッジ）ごとに流量制限が適用されます。

## 3. パイプネットワークのルール

### 3.1 接続の定義

- **一方通行**: すべてのパイプは「出力側」から「入力側」へ流れる一方通行。
- **施設間制限**: 任意の2施設間に結べるパイプは直接1本まで。
- **色分け（フィルタリング）**:
  - 赤、青、黄、緑の4種類の色がある。
  - パイプ自体の色によって、ゲート等での分岐条件に使用される。
  - 1つの施設から異なる色のパイプを複数本伸ばすことが可能。

### 3.2 論理ゲート

- **品質ゲート (Quality Gate)**:
  - 設定された基準（例：品質 80以上）を満たしている液体を「色A」のパイプへ、満たさないものを「色B」のパイプへ自動的に振り分ける。
- **優先ゲート (Priority Gate)**:
  - 「色A」のパイプに空き容量がある限り最優先で流し、色Aが詰まっている（接続先タンクが満杯など）場合にのみ「色B」へ流す。

## 4. 資源の変換

- **液体 → 固体**: 加工施設（攪拌機、圧搾プレス機など）にパイプで牛乳（液体）を供給し、作業を完了させることで固体アイテム（バター、チーズ）を生成する。
- **固体 → 液体**: （将来的な拡張）固体リソースを溶解機等に投入することで、パイプラインへ流す。
