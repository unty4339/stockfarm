# リソースシステム仕様書

本作のリソースは「固体」と「液体」の2つの物理形態に分類され、それぞれ異なる保管・輸送ルールを持ちます。  
固体リソースは本仕様書において **アイテム** と呼称します。

## 1. リソースの分類

### 1.1 固体（アイテム）

- **特徴**: 個別の単位（アイテム）として存在し、タイル上に置かれたりチェストに格納されたりする。
- **輸送**: プレイヤーまたは「運搬」設定の牛が物理的に持ち運ぶ。
- **品質を持つアイテム**: 一部のアイテム（瓶牛乳・バター・チーズ）は品質値を持ち、生成時に使用した液体の品質が引き継がれる。
- **保管**:
  - **タイル上**: 1タイルにつき1アイテムスロットのみ置ける（詳細は「タイル上のアイテム」セクション参照）。
  - **チェスト**: 複数種類のアイテムを同一チェストに混在させることができる。容量は **20個**。

### 1.1.1 アイテム種別とスタック可否

| アイテム名 | スタック可否 | スタック上限 | 備考 |
| --- | --- | --- | --- |
| 栄養ペレット | 不可 | — | 飼料 |
| チーズ | 不可 | — | 品質あり |
| バター | 不可 | — | 品質あり |
| 瓶牛乳 | 不可 | — | 品質あり |
| 小麦 | 可能 | **50個** | — |

### 1.2 液体 (Liquid Resources)

- **特徴**: 連続的な量として存在し、施設間をパイプで移動する。量と「品質」の概念を持つ。
- **輸送**: パイプネットワークを通じて自動的に移動する。
- **例**: 牛乳。
- **保管**:
  - **タンク（設備内蔵）**: 液体を溜める設備内蔵タンク。設備ごとに容量が異なる（詳細は下記）。

#### 設備別タンク容量

| 設備名 | タンク容量 |
| --- | --- |
| 手動搾乳スタンド | 20 L |
| 自動搾乳スタンド | 20 L |
| 足踏み式攪拌機 | 30 L |
| 圧搾プレス機 | 50 L |
| ボトル詰め機 | 50 L |

- 1回の搾乳で得られる牛乳量: **5 L**（搾乳スタンドの内蔵タンクに蓄積される）

## 2. 液体の物理挙動

### 2.1 混合ロジック（平均品質の更新）

タンクに新たに液体 $V_{new}$（品質 $Q_{new}$）が流入すると、タンク内の平均品質 $Q_{avg}$ は以下の式で即座に更新されます。

$$Q_{avg} = \frac{(V_{old} \times Q_{old}) + (V_{new} \times Q_{new})}{V_{old} + V_{new}}$$

- $V_{old}, Q_{old}$: 流入前のタンク内の量と品質。
- タンクが空の場合、最初に流入した液体の品質がそのまま平均品質となります。

### 2.2 流量制限

- パイプライン（施設A→施設B）を流れる液体には流量上限があります。
- **基本流量**: 1 tick あたり 1 単位（設定により調整可能）。
- 複数のパイプが接続されている場合でも、各接続（エッジ）ごとに流量制限が適用されます。

## 3. パイプネットワークのルール

### 3.1 接続の定義

- **一方通行**: すべてのパイプは「出力側」から「入力側」へ流れる一方通行。
- **施設間制限**: 任意の2施設間に結べるパイプは直接1本まで。
- **色分け（フィルタリング）**:
  - 赤、青、黄、緑の4種類の色がある。
  - パイプ自体の色によって、分配器等での分岐条件に使用される。
  - 1つの施設から異なる色のパイプを複数本伸ばすことが可能。

### 3.2 論理分配器 (Distributor)

- **品質分配器 (Quality Distributor)**:
  - 設定された基準（例：品質 80以上）を満たしている液体を「色A」のパイプへ、満たさないものを「色B」のパイプへ自動的に振り分ける。
- **優先分配器 (Priority Distributor)**:
  - 「色A」のパイプに空き容量がある限り最優先で流し、色Aが詰まっている（接続先タンクが満杯など）場合にのみ「色B」へ流す。

## 4. 資源の変換

- **液体 → 固体**: 加工施設（攪拌機、圧搾プレス機、ボトル詰め機など）にパイプで牛乳（液体）を供給し、作業を完了させることで固体アイテム（バター、チーズ、瓶牛乳）を生成する。
  - 生成された固体アイテムは施設内蔵の出力チェストに格納される。チェストが満杯（容量4個）の場合、施設は作動を停止する。
  - 品質を持つ液体（牛乳）から生成される固体（瓶牛乳）は、液体の品質が固体の品質として引き継がれる（混合平均品質の式に従う）。
- **固体 → 液体**: （将来的な拡張）固体リソースを溶解機等に投入することで、パイプラインへ流す。

## 5. タイル上のアイテム

### 5.1 配置ルール

- アイテムはタイル上に置くことができる
- **1タイルにつき1アイテムスロット**のみ配置できる（異なる種類の混在不可）
- 設備が配置されているタイルにはアイテムを置けない

### 5.2 スタック

- 一部のアイテムは1スロット内でスタック（複数個積み重ね）できる
- スタック上限はアイテム種別ごとに決まっている（詳細は「アイテム種別とスタック可否」の表を参照）
- 品質を持つアイテムは個体ごとに品質が異なるため、スタック不可

### 5.3 アイテムを落とすときのアルゴリズム

あるタイルにアイテムを落とすとき、以下のルールで配置先を決定する。

#### 非スタック可能アイテムの場合

1. 対象タイルが空き（アイテムなし・設備なし）→ そこに配置して終了
2. 占有済み → 時計回りに隣接する8タイルを順番に探索し、最初の空きタイルに配置

#### スタック可能アイテムの場合

1. 対象タイルに**同種アイテムが存在しスタック未満** → スタックに追加
   - 上限を超えた余剰分は、時計回りに隣接する空きタイルを探して配置（以降繰り返し）
2. 対象タイルが**空き** → 新規配置して終了
3. 対象タイルが**占有済み**（設備あり・別アイテムあり・スタック満杯） → 時計回りに隣接する空きタイルを探して配置

#### 時計回りの方向順

上 → 右上 → 右 → 右下 → 下 → 左下 → 左 → 左上

#### 空きタイルが見つからない場合

隣接8タイルが全て埋まっている場合はアイテムを破棄する（将来的に探索範囲を拡大予定）。

## 6. 廃棄物（糞尿）の処理

廃棄物はアイテム（固体リソース）ではなく、タイルに付与される「汚染マーカー」として扱います。

- 牛が設備でのタスクを完了するたびに、その設備の隣接タイルに廃棄物マーカーが1つ生成されます
- 廃棄物マーカーは牛または牧場主の清掃タスクによって消滅します（チェストへの運搬は不要）
- 廃棄物マーカーは売却できません
- 廃棄物マーカーが存在するタイルが部屋内にある場合、そのマーカー1個につきムードが以下の通り低下します

| ムード種別 | 1個あたりの低下量 |
| --- | --- |
| 作業ムード | -5 |
| 休憩ムード | -5 |
| 繁殖ムード | 0 |
