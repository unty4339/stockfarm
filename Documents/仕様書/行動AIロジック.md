# ワーカー牛：行動AIロジック詳細仕様

本作のAIは、時間経過（朝昼晩のサイクル）と、24時間のスケジュール設定に基づいた意思決定を行い、その結果を「タスク」として実行します。

## 1. 時間経過とスケジュールシステム

### 1.1 時間サイクル

ゲーム内時間は24区分（スロット）に分けられ、視覚的なサイクルと連動します。

- **朝 (Morning)**: スロット 4 〜 7
- **昼 (Day)**: スロット 8 〜 19
- **夜 (Night)**: スロット 20 〜 3

### 1.2 スケジュール設定

各ワーカーは24個のスロットそれぞれに対して、以下の3つの「スケジュールモード」のいずれかを割り当てられます。

| モード | 説明 | 優先される振る舞い |
| --- | --- | --- |
| **労働 (Work)** | 牧場運営を優先する時間 | 「加工」「運搬」のタスクを積極的に探し、実行する。 |
| **娯楽 (Joy)** | 幸福度を維持する時間 | 「慰安」「搾乳」を優先。必要に応じて「休憩」も行う。 |
| **就寝 (Sleep)** | 体力を回復する時間 | 「休憩（睡眠）」を最優先。空腹でない限り、他の作業は行わない。 |

### 1.3 時間単位

| 定義 | 値 |
| --- | --- |
| 1ゲーム内日 | 600 tick |
| 1倍速時の換算 | 1 tick = 1 秒（1日 = 10 分） |
| ゲーム速度倍率 | ×1〜×3 をプレイヤーが設定可能 |

## 2. タスク (Task) の定義

すべてのタスクは「出産」「休憩」「搾乳」「慰安」「加工」「運搬」のいずれかの作業分類に属します。

| タスク名 | 作業分類 | 発生トリガー | 目標地点 | 完了条件 |
| --- | --- | --- | --- | --- |
| **出産** | 出産 | 妊娠経過 100 | 現在地 または 柔らかい床/ベッド | 子牛の生成 |
| **睡眠** | 休憩 | スタミナ低下時 または 就寝時間 | 割り当てベッド → 未割り当ての空きベッド（自動紐づけ）→ 現在地 | スタミナ回復 |
| **摂食** | 休憩 | 空腹度 < 20 | 最寄りの給餌桶 または 牧草 | 空腹度 100 |
| **搾乳待機** | 搾乳 | 乳蓄積量 100 | 空きのある搾乳スタンド | 搾乳作業の完了 |
| **ケア利用** | 慰安 | 幸福度 < 40 または 娯楽時間 | 空きのある自動ブラシ機 | 幸福度回復 |
| **加工補助** | 加工 | 作業設定ON 且つ 施設空きあり | 対象施設の操作マス | 作業完了 |
| **運搬 (Hauling)** | 運搬 | 運搬設定ON 且つ 優先度上位チェストへの未実行運搬タスクが存在 | 出発チェスト → 到着チェスト | 納品完了 |
| **清掃** | 清掃・整理 | 作業設定ON 且つ 廃棄物マーカーが存在 | 廃棄物マーカーのあるタイル | タイルの廃棄物マーカーを消滅させる |
| **アイドル散歩** | なし | 他に優先タスクなし | 所属ゾーン内のランダム座標 | 到着 |

## 3. 意思決定の優先順位 (Decision Hierarchy)

思考サイクルごとに、以下のルールに従って実行すべきタスクを決定します。

### 3.1 優先順位の決定フロー

1. **緊急トリガー**: 「出産」タスクはスケジュールに関わらず最優先。
2. **現在のスケジュールモードの確認**:
   - **就寝モード**: 「休憩（睡眠・摂食）」以外のタスク（出産除く）を原則として無視する。
   - **労働モード**: 「加工」「運搬」のタスクをアクティブにする。
   - **娯楽モード**: 「慰安」「搾乳」をアクティブにし、「加工」「運搬」を後回しにする。
3. **個別優先度の適用**: プレイヤーが設定した作業分類ごとの優先度（1〜7）を確認。
4. **同一優先度時のデフォルト順序**:
   1. **出産** > 2. **休憩** > 3. **搾乳** > 4. **慰安** > 5. **加工** > 6. **運搬** > 7. **清掃・整理**

## 4. タスク実行のライフサイクル

### 4.1 通常タスクのフェーズ

1. **予約 (Reservation)**: ターゲット（施設のアサイン枠やチェスト）をロックする。
2. **移動 (Moving)**: A*アルゴリズムによりパスを生成し、目標地点へ移動。
3. **実行 (Execution)**: 到達後、固有のアクションを開始。
4. **完了・破棄 (Finishing)**: 完了条件を満たした場合、予約を解除。

### 4.2 運搬タスクの特殊フロー

運搬は「出発」と「到着」の2つの目標地点を持つ多段階フローです。

1. **取出フェーズ**: 「出発チェスト」へ移動し、アイテムを「保持スロット」へ。
2. **移動フェーズ**: アイテムを保持した状態で「到着チェスト」へ。
3. **納品フェーズ**: 「到着チェスト」にアイテムを入れ、タスク完了。

## 5. 移動と経路探索 (Movement & Pathfinding)

### 5.1 移動コスト

- `床`: 移動コスト 1 / `地面`: 2 / `ゲート`: 2 / `柵・壁`: ∞（通行不可）

### 5.2 ゾーンの定義

- **ゾーン（Zone）**: プレイヤーが牧場マップ上に手動で設定した名前付きエリアです。
- ゾーンは任意のグリッド範囲をペイントするように指定でき、複数のゾーンが重複してもかまいません。
- 各牛はプレイヤーによって1つのゾーンに所属するよう割り当てられます。
- 牛はアイドル時や移動先の指定がない場合、所属ゾーン内にとどまろうとします（ゾーン外への移動コストが大幅に増加）。
- ゾーン未割り当ての牛は、牧場全体をゾーンとして扱います。

### 5.3 動的再検索

ターゲット消失時やゲート閉鎖時は、即座に再検索または待機する。

## 6. 作業の中断ロジック

タスク実行中に以下の割り込みが発生した場合、進行度を保存してタスクを破棄します。

- プレイヤーによる「強制移動」命令。
- スケジュールモードが切り替わり、現在のタスクが非推奨となった（例：労働時間終了で就寝時間になった）。
- 設定優先度に基づき、より上位のタスク条件が発生した。

## 7. プレイヤーによる介入 (Forced Command)

プレイヤーがワーカーを直接選択して施設や場所を指定した場合、生成された「強制タスク」は、スケジュールの制限を無視して「完了」するまで最優先で実行されます。

種付け施設への誘導もこの強制タスクとして扱います。プレイヤーが牛を選択して「種付け施設へ移動」を指示すると、牛は施設の牛スロットへ向かい、プレイヤーの操作を待機します。

## 8. チェスト優先度と自動運搬システム

### 8.1 チェストの設定パラメータ

各チェストはプレイヤーが以下のパラメータをUIから設定できます。

| パラメータ | 説明 |
| --- | --- |
| **優先度** | 整数値（大きいほど高優先度）。初期値は 0 |
| **フィルター条件** | このチェストが受け入れるアイテム種別。未設定の場合はすべてのアイテムを受け入れる |

### 8.2 自動運搬タスクの生成ロジック

以下のいずれかのイベントが発生した際に、自動運搬タスクの生成を試みます。

**トリガーイベント:**
- あるチェストにアイテムが追加されたとき（運搬による納品・加工施設からの出力など）

**生成手順:**

1. アイテムが追加されたチェストの優先度 $P$ を確認する
2. 以下をすべて満たす「候補チェスト」を全チェストから検索する:
   - 優先度が $P$ より厳密に大きい
   - フィルター条件がそのアイテム種別を受け入れる
   - 空き容量がある
3. 候補が1つ以上ある場合:
   - 最も高い優先度を持つチェストを選択する
   - 最高優先度が同値のチェストが複数ある場合は、最短経路のチェストを選択する
4. 選択されたチェストへの運搬タスクを予約し、出発チェストの該当アイテムをロックする

**生成しない条件:**
- 候補チェストが存在しない場合（上位優先度なし・フィルター不一致・満杯）
- 同じ優先度のチェスト間では運搬タスクを生成しない

### 8.3 運搬タスクのキャンセル

予約済み運搬タスクは以下の場合にキャンセルされます。

- 到着チェストが他の運搬で満杯になった場合
- 出発チェストのアイテムが別の手段で消費・移動された場合

キャンセル後は、出発チェストのアイテムに対して §8.2 の生成ロジックを再実行します。
