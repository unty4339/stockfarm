## AITaskBase

```
abstract class
```

全タスクの基底クラス。タスクのライフサイクル（予約→移動→実行→完了）を管理する。

### プロパティ

| 名前 | 型 | 説明 |
|---|---|---|
| `Owner` | `WorkerBase` | タスクを実行するワーカー |
| `Priority` | `int` | タスク優先度（1〜7、低い値が高優先） |
| `State` | `AITaskState` | 現在の実行状態 |
| `TargetPosition` | `Vector2Int` | 移動目標座標 |

### メソッド

```csharp
// このタスクが現在実行可能かを判定する
abstract bool CanExecute()

// タスク固有の実行ロジック（Executing状態のtickごとに呼ばれる）
abstract void OnExecute()

// タスクが完了したかを判定する
abstract bool IsComplete()

// タスクを開始する（Pending → Moving or Executing に遷移）
void Start()

// 毎tick呼ばれる更新処理（状態遷移を内部で行う）
void Tick()

// タスクを中断する（→ Interrupted に遷移）
void Interrupt()
```

## AITaskState (enum)

```csharp
enum AITaskState
{
    Pending,      // 予約済み・未開始
    Moving,       // 目標地点へ移動中
    Executing,    // 実行中
    Completed,    // 完了
    Interrupted,  // 中断
}
```

## 具体タスク

### BirthTask : AITaskBase

出産タスク。最優先（Priority = 0）で発動し、スケジュールを無視する。

```csharp
// 所有者: CowWorker
// 完了条件: CowWorker.GiveBirth() の呼び出し完了
// CanExecute: Owner.HasEffect(StatusEffectType.PregnancyLate) かつ出産タイミング到達
```

### SleepTask : AITaskBase

睡眠タスク。割り当て済みのベッドに向かい、スタミナを回復する。

```csharp
// 完了条件: スタミナ 100 に到達、またはスケジュール変更
// CanExecute: スケジュールがSleep、または Fatigue 状態
// 使用設備: CowBed（AssignedCow と Owner が一致するもの）
```

### EatTask : AITaskBase

摂食タスク。給餌桶または食料棚へ移動して食料を消費する。

```csharp
// 完了条件: Hunger が閾値以下に低下
// CanExecute: Owner.HasEffect(StatusEffectType.Hungry)
// 使用設備: FeedingTrough（牛）または FoodShelf（牧場主）
// コンストラクタ引数: EquipmentBase targetFood
EatTask(WorkerBase owner, EquipmentBase targetFood)
```

### MilkWaitTask : AITaskBase

搾乳待機タスク。搾乳スタンドへ移動し、搾乳操作を待機する。

```csharp
// 完了条件: 搾乳完了
// CanExecute: Owner は CowWorker、Lactating状態、MilkAccumulation >= 閾値
// 使用設備: ManualMilkingStand または AutoMilkingStand
```

### CareTask : AITaskBase

ケア利用タスク。自動ブラシ機を使用して幸福度を向上させる。

```csharp
// 完了条件: ケア1回分の時間が経過
// CanExecute: AutoBrush が利用可能かつスケジュールがJoy
```

### ProcessingAssistTask : AITaskBase

加工補助タスク。加工設備に補助要員として参加する。

```csharp
// 完了条件: 加工設備の1サイクル完了
// CanExecute: 補助可能な ProcessingEquipmentBase が存在する
// 使用設備: ProcessingEquipmentBase の派生クラス
```

### TransportTask : AITaskBase

運搬タスク。2フェーズ（取出→納品）を内部状態で管理する。

```csharp
// フェーズ1: Source から ResourceBase を取り出す
// フェーズ2: Destination へ ResourceBase を納品する
// 完了条件: 納品完了
// CanExecute: Source に指定リソースが存在する
// コンストラクタ引数:
TransportTask(WorkerBase owner, IContainer source, IContainer destination, ResourceType resourceType, int amount)

// 内部フェーズ管理
private enum TransportPhase { TakingOut, Delivering }
```

### CleanTask : AITaskBase

清掃タスク。廃棄物マーカーのあるタイルへ移動して除去する。

```csharp
// 完了条件: 対象WasteMarker の Severity が 0 に低下
// CanExecute: マップ上にWasteMarkerが存在する
// コンストラクタ引数:
CleanTask(WorkerBase owner, WasteMarker target)
```

### IdleWalkTask : AITaskBase

アイドル散歩タスク。やることがない時にランダムな近隣タイルを歩く。

```csharp
// 完了条件: 目標タイルに到達
// CanExecute: 常にtrue（他に実行可能タスクがない場合に選択される）
```

## AIDecisionMaker

```
class（WorkerBase に1対1で保持される）
```

タスクの優先度評価を行い、実行すべきタスクを決定する。

### プロパティ

| 名前 | 型 | 説明 |
|---|---|---|
| `Owner` | `WorkerBase` | 管理対象のワーカー |
| `TaskPriorities` | `Dictionary<Type, int>` | タスク種別ごとの優先度設定（1〜7） |

### メソッド

```csharp
// 現在の状態に基づき最適なタスクを決定して返す
// 優先順序: BirthTask → スケジュール確認 → 個別優先度 → デフォルト順
AITaskBase DecideTask()

// プレイヤー命令によるタスクを強制設定する（スケジュール制限を無視）
void ForceTask(AITaskBase task)

// タスクの優先度を設定する（UIからの操作に対応）
void SetTaskPriority(Type taskType, int priority)
```

### 意思決定の優先順序

1. BirthTask（最優先、PregnancyLate状態のとき）
2. 現在スロットのスケジュール種別（Sleep/Work/Joy）に合致するタスク
3. 個別優先度（TaskPriorities）の低い値順
4. デフォルト順（IdleWalkTask が最後尾）

## IPathCostProvider (interface)

A*経路探索でのタイルコスト提供インタフェース。

```csharp
interface IPathCostProvider
{
    // 指定タイルの移動コストを返す（通行不可の場合は int.MaxValue を返す）
    int GetCost(Vector2Int position)
}
```

### 標準コスト値

| タイル種別 | コスト |
|---|---|
| 床（Floor） | 1 |
| 地面（Ground） | 2 |
| ゲート（Gate） | 2 |
| 壁（Wall） | int.MaxValue（通行不可） |
| 柵（Fence） | int.MaxValue（通行不可） |

## PathFinder (static)

A*アルゴリズムを使用した経路探索クラス。

```csharp
static class PathFinder
{
    // from から to までの経路をグリッド座標リストで返す
    // 経路が存在しない場合は空のリストを返す
    static List<Vector2Int> FindPath(
        Vector2Int from,
        Vector2Int to,
        IPathCostProvider costProvider)
}
```
